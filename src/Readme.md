# Тестирование API сервиса заказа самокатов
## Тестируем ручки
Проверь, что они корректно работают и выдают нужные ошибки. 
### Создание курьера 
Проверь:  
`[passed]`курьера можно создать;\
`[passed]`нельзя создать двух одинаковых курьеров;\
`[passed]`чтобы создать курьера, нужно передать в ручку все обязательные поля;\
`[passed]`запрос возвращает правильный код ответа;\
`[passed]`успешный запрос возвращает ok: true;\
`[passed]`если одного из полей нет, запрос возвращает ошибку;\
`[passed]` если создать пользователя с логином, который уже есть, возвращается ошибка.\
### Логин курьера
Проверь:\
`[passed]` курьер может авторизоваться;\
`[bug scooter-1]` для авторизации нужно передать все обязательные поля;\
`[passed]` система вернёт ошибку, если неправильно указать логин или пароль;\
`[bug scooter-1]` если какого-то поля нет, запрос возвращает ошибку;\
`[passed]` если авторизоваться под несуществующим пользователем, запрос возвращает ошибку;\
`[passed]` успешный запрос возвращает id.
### Создание заказа
Проверь, что когда создаёшь заказ:
* можно указать один из цветов — BLACK или GREY;
* можно указать оба цвета;
* можно совсем не указывать цвет;
* тело ответа содержит track.
* Чтобы протестировать создание заказа, нужно использовать параметризацию.
### Список заказов
Проверь, что в тело ответа возвращается список заказов.
